**Project Title:** Mac Hive Corporate Website & Portfolio
**Version:** 1.0
**Date:** February 8, 2026
**Author:** Mohammed Haris

#### **1. Executive Summary**

Mac Hive is transitioning from a personal portfolio to a professional corporate entity. The new platform serves a dual purpose:

1. **SaaS Publisher:** A central hub to showcase and support proprietary products (BillHive, AdHive, and future influencer marketplaces).
2. **Service Provider:** A client-facing portal offering high-speed web and mobile development services. The unique selling proposition (USP) is the "Agentic Workflow"â€”leveraging AI-assisted architecture to reduce development timelines from months to weeks.

#### **2. Technical Specifications & Constraints**

**Core Stack:**

* **Web Framework:** Next.js (App Router)
* **Language:** TypeScript
* **Styling:** Tailwind CSS + Shadcn/UI (for rapid, consistent components)
* **Mobile Services Stack:** Flutter
* **Hosting:** Vercel
* **Version Control:** GitHub

**Agentic Workflow Tools:**

* **Design Generation:** Google Stitch (for rapid UI iteration)
* **Code Generation:** Anti-gravity, Cursor, VS Code Copilot

#### **3. Functional Requirements**

**3.1. Landing Page (Home)**

* **Objective:** Establish trust immediately. Visitors must recognize Mac Hive as a professional agency, not a freelancer.
* **Key Message:** "Enterprise quality at startup speed."
* **Hero Section:** must highlight the "Agentic Workflow" advantage.

**3.2. Product Showcase (SaaS Portfolio)**

* **Requirement:** Dynamic card-based layout to display internal products.
* **Current Products:**
* **BillHive:** Mobile billing application.
* **AdHive:** SaaS application for advertising/influencers.


* **Features:**
* Status badges (Live, Beta, In Development).
* Direct links to product sub-domains or download pages.



**3.3. Services Module**

* **Web Development:**
* Focus on Next.js best practices.
* Emphasis on performance and SEO.


* **Mobile Development:**
* Focus on Flutter for cross-platform delivery.


* **Workflow Explanation:** A dedicated section explaining how the "Design (Stitch)  Code (Agentic)  Deploy" process saves the client time and money.

**3.4. Client Acquisition & Support**

* **Inquiry Form:** Smart form separating "Service Inquiries" (New Clients) from "Product Support" (Existing Users).
* **Global Reach:** Content must be localized for an international audience (neutral English, global standard terminology).

#### **4. Non-Functional Requirements**

* **Performance:** 95+ Score on Google PageSpeed Insights.
* **Design aesthetic:** Pastel color palette, clean lines, high readability.
* **Responsiveness:** Mobile-first design approach.
